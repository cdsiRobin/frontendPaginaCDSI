SISTEMA DE ADMINISTRACION DE LEGAJOS DE PERSONAL PARA ENTIDADES PUBLICAS Y PRIVADAS

“SILEG v. 2.4”

1. GENERALIDADES DEL SISTEMA
  • Los objetivos del “Sistema de Administración de Legajos de Personal - SILEG” ,son: Crear una base de datos con información actualizada, completa y confiable, teniendo como fuente el legajo de personal (incluye documentos que sustentan) y/o la ficha de datos (manual o Computarizada) de cada trabajador activo, cesantes D.L. 20530, o Jubilado D.L.19990, fallecido (tendrá beneficiarios) y pasivo o retirado.
  • Permitir el Registro y mantenimiento de todos los datos consignados en la nueva ficha de datos.
  • Permitir la explotación básica de las bases de datos hasta obtener la Constancia Escalafonaria que es un subproducto de uso cotidiano en cada sede administrativa de un Ministerio.
  • Facilitar la emisión de Inconsistencias para la verificación de datos.
  • Permitir el control computarizado de la digitación.
  • Facilitar la toma de decisiones con una serie de consultas.
  • Permitir la emisión de listados y reportes de control en los distintos pasos del proceso.
  • Efectuar el mantenimiento del aplicativo: Proceso que permite al usuario realizar una serie de operaciones destinadas a efectuar los respaldos y medidas de seguridad y actualización de los archivos y tablas que dan soporte a la operatividad del sistema.

El sistema “SILEG” completo lo utilizamos para hacer el servicio de Consultoría llevando diferentes controles que no son necesarios en la operatividad de las Oficinas de Escalafón para el ingreso interactivo (on-line) de los legajos de los trabajadores del Ministerio de Educación, por ello lo hemos personalizado con las opciones mínimas necesarias.

La oficina de Escalafón de las Direcciones Regionales de Educación, Subdirecciones Regionales de Educación y Direcciones de Servicios Educativos, con el presente Sistema aplicativo dispondrán de una gran herramienta para la optimización de sus servicios en el área de Escalafón (Legajos); lo que le permitirá racionalizar sus recursos, minimizar sus costos y fundamentalmente brindar un servicio eficiente. Para lograr lo anterior se recomienda mantener actualizada todos los registros de personal es decir ingresar los movimientos al sistema SILEG, en el mismo momento que se está produciendo la transacción administrativa.

4. REQUERIMIENTOS DE EQUIPOS DE COMPUTO Y SOFTWARE BASE PARA FUNCIONAMIENTO DEL SISTEMA “SILEG”
  • 4.1 REQUERIMIENTO DE EQUIPOS DE COMPUTO (HARDWARE)

    • El Sistema de Administración de LEGAJOS DE PERSONAL “SILEG” puede correr en una red o en un ambiente monousuario la configuración mínima de los equipos para el funcionamiento de este sistema es la siguiente:

    • 4.1.1 SERVIDOR (Configuración mínima)
      • PROCESADOR: Intel(R) Xeon(R) CPU E3-1245
      • MEMORIA RAM: 16GB
      • DISCO DURO: 1T
      • MOUSE
      • ESTABILIZADOR DE VOLTAJE
      • LECTOR DE CD
    • 4.1.2 ESTACIONES DE TRABAJO (Configuración mínima)
      • PROCESADOR: Intel® Core™ i3,i5,i7 y AMD
      • MEMORIA RAM: 4GB
      • DISCO DURO: 500GB
      • MOUSE
      • ESTABILIZADOR DE VOLTAJE
      • LECTOR DE CD
    • 4.1.3 IMPRESORAS.
      • CUALQUIER TIPO
    • 4.1.4 RECOMENDACIÓN

        • La configuración mencionada es mínima; sin embargo lo recomendable es que las estaciones de trabajo sean equipos Windows 7,8 y 10 de 4 MB de RAM, e impresora láser o Inyección de Tinta ; con lo que se logrará un mejor perfomance, ya que el sistema trabaja en ambiente Windows.

  • 4.2 CARACTERISTICAS DE PROGRAMACION (SOFTWARE)
    • 4.2.1 SISTEMA OPERATIVO
      • 4.2.1.1 PARA RED

        • Novell Netware 4.x ó Windows NT, o red Windows 98, Windows Server 2000, Windows XP correctamente instalado y configurado.

      • 4.2.1.2 PARA LA ESTACION DE TRABAJO

        • Windows 98 o superior, correctamente instalado y configurado.

    • 4.2.2 LENGUAJE DE PROGRAMACION:
      • El lenguaje de programación utilizado en este sistema, es VISUAL FOXPRO versión 6.0. Trabaja en modo multiusuario o monousuario (solo en una PC).
      • No necesita el compilador del Foxpro para ejecutar el sistema toda vez que el sistema SILEG esta en modo ejecutable (SILEG.EXE).
5. ADECUACIONES PARA LA PERSONALIZAR A LA REALIDAD DE CADA ENTIDAD O EMPRESA

CDSI, podrá efectuar las adecuaciones para los casos siguientes:

  • Una vez que terminaron depurara todos los datos del legajo de los trabajadores, se podrá excluir las opciones de los procesos batch y solamente activar los procesos interactivos para que quede una solución optima en tiempo real.
  • Se podrá adecuar y/o hacer desarrollos complementarios para llegar a tener una solución ala medida de las necesidades de cada entidad pública o privada..
